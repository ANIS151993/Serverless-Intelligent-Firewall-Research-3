Scientific Positioning

Research-3 is framed as a distinct contribution built from Research-1 and Research-2, not a copy. The novelty is architectural, methodological, and operational.

Enterprise Multi-Tenant Model

Super-control governance provisions corporate clients while each tenant runs a compact subsystem with resilient local protection and synchronized upgrades.

Continuous Threat Adaptation

OSINT feeds are fused with runtime telemetry to support drift-aware adaptation and better handling of emerging threats and zero-day signals.

Key Journal Contributions

  • Unified ASLF-OSINT architecture with super-control and tenant-local subsystems.
  • REST API-driven intelligence ingestion from MISP, OTX, and VirusTotal.
  • Multi-paradigm learning stack: PPO, continual adaptation, meta-learning, transfer, federated aggregation.
  • Real-time role-aware operations through authenticated control APIs and dashboard streaming.

Journal Summary

Everything aligned to your manuscript

Abstract snapshot

ASLF-OSINT addresses evolving cloud-native threats by combining live threat intelligence, adaptive learning, and federated zero-trust governance. The system demonstrates strong detection quality while maintaining low latency and multi-tenant operational control.

Method stack

  • PPO policy optimization
  • Continual drift adaptation
  • Few-shot meta-learning
  • Transfer learning
  • FedNova-based federated updates

Data and runtime scope

  • CICIDS2017, CSE-CIC-IDS2018, UNSW-NB15, NSL-KDD
  • 30-day live OSINT ingestion
  • AWS Lambda, Azure Functions, GCP Functions
  • Proxmox federation node deployment path
